# CLAUDE.md

## Rules

- Follow a plan, if exists, strictly. Do not deviate from it.
- Do not implement tasks that are beyond the scope of your plan
- If you are not sure, stop and say "I'm not sure about this task"
- Do not overcomplicate stuff. Simple/Lean approach is better than complex one
- Write semantic and idiomatic code. Use clear and descriptive names for variables, functions, and classes.
- Prefer readability over complexity. Write code that is easy to understand and maintain.
- Find and create solutions based on the complexity of the task. Find simple solutions for simple problems. If the problem is complex, explore solutions and create a plan.
- Always validate your work either through tests, sample ui, running a bash command, etc. Never skip any review

## Workflow

> If a task is complex, follow this workflow

1. Explore the codebase and solutions to the problem
2. Create a plan based on findings and exploration
3. Implement the plan. Assess if we need tests. If yes, always default to TDD(Test Driven Development)
4. Verify your work either through tests, screenshots, running the server, console logs, etc
5. Report your work to the user

> If a task is simple enough

1. Create tasks/To Do list to track your work
2. Validate your work
3. Report your work
